关卡 0：认识 GitHub
▶

知识要点

GitHub = 代码的"云端项目文件夹"。它的本质和网盘差不多——帮你存文件、记录改动。但比网盘更聪明：能追踪"谁在什么时候改了什么"，改坏了能一键回滚。

GitHub 上线于 2008 年，2018 年被微软以 75 亿美元收购。现在托管超过 4 亿个代码仓库，1 亿+ 活跃用户。引爆了开源文化——Python、React、安卓等知名项目都在上面。

打卡任务

  • ✓
    阅读知识要点，心里复述一遍"GitHub 是什么"
  • ✓
    打开 github.com，看一眼首页，感受界面
✅ 打卡确认：能说出 GitHub 的三个主要作用（存代码、记改动、协作）

关卡 1：搞懂最核心的四个概念
▶

知识要点

GitHub 有很多术语，但下面 4 个是骨架，弄懂了就懂了一半：

  • Repository（仓库，Repo）：项目文件夹，分 Public（公开）和 Private（私密）
  • Branch（分支）：从主线复制出来的"实验线"，改坏了不影响主线。main = 正式版，branch = 草稿
  • Commit（提交）：每完成一轮修改，按一次"保存"并附上说明，类似带备注的存档
  • Pull Request（PR）："请把我的改动合并进主线"，发起申请让维护者审核

打卡任务

  • ✓
    在脑海里复述：Repo / Branch / Commit / PR 是什么意思
  • ✓
    用一句话向自己解释这四个概念（讲给自己听）
✅ 打卡确认：关卡 0 的任务全部打勾

关卡 2：注册账号 + 建第一个 Repo
▶

知识要点

打开 github.com/signup 免费注册。注册完成后：

  1. 点击右上角 + → New repository
  2. Repository name 填写：hello-world
  3. 选择 Public（公开，让全世界看到你的进步）
  4. 勾选 Add a README file（自动生成说明文件）
  5. 点击 Create repository

就这样，你有了第一个 GitHub 仓库。

打卡任务

  • ✓
    注册 GitHub 账号（如已有，跳过）
  • ✓
    创建名为 hello-world 的公开 Repo
  • ✓
    在 Repo 页面找到并点击 commits，感受改动记录
✅ 打卡确认：在 github.com 上能看到自己的 hello-world 仓库，commits 页面显示至少 1 条提交记录

关卡 3：编辑文件，理解 Commit
▶

知识要点

进入 hello-world Repo，点击 README.md 文件上的 Edit（铅笔图标），修改内容：

# Hello World

这是我在 GitHub 上的第一个仓库！
今天开始学习 GitHub — 2026年4月25日。

改完后页面会提示 Commit changes，在 commit 框里写备注，如：添加了自我介绍和日期，然后点击 Commit changes。

这就是一次完整的 GitHub 操作：编辑 → 写备注 → 提交。

打卡任务

  • ✓
    提交时写一句有意义的 commit 信息（不是默认的"Update README.md"）
  • ✓
    点进 commits 页面，找到自己的 commit，确认信息正确
✅ 打卡确认：commits 历史里可以看到自己的 commit，备注清晰可读

关卡 4：理解 Branch（分支）
▶

知识要点

Branch 就是解决"想改又怕改乱"问题的工具——先复制一份出来，在副本里改，改好了再合并回来。

在 GitHub 上，默认分支叫 main（主线/稳定版）。

点击 Repo 里的 <> Code 按钮，旁边就是 main 下拉菜单——这就是切换分支的地方。

  1. 点 main 下拉菜单 → 输入新分支名（如 test-branch）→ 点击 Create branch
  2. 你就拥有了一条新分支，可以随便折腾，不影响 main

打卡任务

  • ✓
    在 hello-world Repo 里新建一条分支，名字叫 test-branch
  • ✓
    切换到 test-branch，修改 README.md，加一行："这是我分支里的实验内容"
  • ✓
    提交这次修改（commit message 写：在 test-branch 里实验）
  • ✓
    切回 main 分支，确认 README.md 内容没有变化
✅ 打卡确认：能看到 main 和 test-branch 各有一版内容，互不干扰

关卡 5：发起第一个 Pull Request
▶

知识要点

PR = Pull Request，意思是"请把我的改动拉进来"。当你确认分支里的改动没问题了，就可以发起 PR，请求合并到 main。

操作步骤：

  1. 切换到 test-branch
  2. 点击上方的 Contribute 按钮 → Open pull request
  3. 标题自动填充，可以不改
  4. 底部点击 Create pull request

PR 页面会显示：改了哪些文件、多了哪几行、少了哪几行，一目了然。

打卡任务

  • ✓
    在 hello-world Repo 里，从 test-branch 发起一个 Pull Request
  • ✓
    在 PR 页面查看 changed files，感受"改动一览"的感觉
  • ✓
    合并这个 PR（点击 Merge → Confirm merge）
  • ✓
    合并后切到 main 分支，确认 README.md 已更新为分支内容
✅ 打卡确认：PR 已合并，main 分支的 README.md 包含了分支里的修改内容

关卡 6：理解 GitHub 的社交属性
▶

知识要点

  • Star（收藏）：点个星，像给微博点赞。Star 数量是衡量项目热门程度的指标
  • Fork（派生）：把别人的仓库复制一份到自己的账号下，可以随便改，不影响原项目。这是参与开源的最简单方式
  • Watch（关注）：Watch 一个仓库后，它有更新你会收到通知，适合跟踪你关心的项目
  • Explore：github.com/explore 是发现页，里面有 Trending（热门项目）和各种主题合集

打卡任务

  • ✓
    去 github.com/explore 逛逛，看看 Trending 页面
  • ✓
    找一个感兴趣的项目，点进去看看（感受界面，不用做任何操作）
  • ✓
    随手 Star 1-2 个觉得有意思的仓库
  • ✓
    了解 Fork：在任意一个公开仓库页面，找到 Fork 按钮
✅ 打卡确认：能在 Explore 找到感兴趣的项目，说出 Star / Fork / Watch 三者的区别

关卡 7：用 gh CLI 连接 GitHub
▶

知识要点

你已经安装了 gh（GitHub CLI），现在来完成初始化授权：

gh auth login

按提示操作：选择 GitHub.com → 选择 HTTPS → 浏览器弹出授权确认 → 命令行自动继续。

验证安装成功：

gh auth status        # 查看登录状态
gh repo list          # 列出你的仓库

克隆你之前建的 hello-world：

gh repo clone 你的GitHub用户名/hello-world

打卡任务

  • ✓
    运行 gh auth login 完成授权
  • ✓
    运行 gh auth status 确认已登录
  • ✓
    运行 gh repo clone 你的用户名/hello-world 克隆到本地
  • ✓
    在克隆下来的文件夹里，cat README.md 查看内容
✅ 打卡确认：gh auth status 显示 logged in，克隆的文件夹里能看到 README.md 内容

关卡 8：用 gh 做一次完整的提交推送
▶

知识要点

现在你在本地有了一个仓库，来做一次完整的 Git 操作流程：

cd hello-world                      # 进入仓库目录
echo "新增加一行测试内容" >> README.md   # 在文件末尾追加一行
git add .                         # 打包所有改动
git commit -m "添加测试内容"       # 提交，附备注
git push                          # 推送到 GitHub 云端

解释：git add . = 打包所有改动 → git commit -m "备注" = 正式存档 → git push = 推送到云端。

然后去网页版刷新 hello-world，看 README.md 是否已更新。

打卡任务

  • ✓
    在本地仓库里新增或修改任意内容
  • ✓
    完整执行 add → commit → push 全流程
  • ✓
    在 github.com 网页上刷新，确认改动已同步上去
✅ 打卡确认：网页上的 README.md 显示了你刚 push 的内容

关卡 9：认识 Issues
▶

知识要点

Issue = 问题/任务跟踪系统。开源项目用它来：

  • 记录 Bug（有人发现问题了，开一个 Issue）
  • 提需求（有人建议加新功能）
  • 做任务分配（维护者把任务列成 Issue）

GitHub Issues 界面类似 Todo 列表，每个 Issue 有：标题、正文描述、状态（Open / Closed）、评论。

打卡任务

  • ✓
    在 hello-world Repo 里新建一个 Issue，标题：测试 Issue，内容：这是我创建的第二个 GitHub Issue
  • ✓
    给这个 Issue 加一个 Label，点击 Labels → 选择 enhancement
  • ✓
    Close 这个 Issue（因为只是测试）
✅ 打卡确认：在 Issues 页面能看到已关闭的"测试 Issue"
